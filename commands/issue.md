# issue — rocky 개선 아이디어/버그를 이슈로 캡처

다른 레포에서 rocky 를 쓰다 떠오른 **기능 제안**이나 **버그**를, 지금 작업 흐름을 끊지 않고
`minjun0219/rocky` 의 GitHub Issue 로 남긴다. `$ARGUMENTS` 는 아이디어/버그 요지(있으면).
출력·이슈 본문은 **한국어** (코드 identifier / 경로 / 명령어 / API path 는 영어 그대로).

## 원칙

1. **타깃은 항상 `minjun0219/rocky`.** 이 커맨드는 *다른* 레포에서 부르는 게 목적이라 현재
   레포의 remote 를 믿지 않는다 — `gh` 호출마다 `--repo minjun0219/rocky` 를 명시한다.
2. **rocky 는 토큰을 직접 다루지 않는다.** 이슈 생성은 전부 `gh` CLI 에 위임한다 (`/finish` 와
   동일 정책).
3. **생성 전 한 번 확인.** GitHub 이슈는 외부 산출물이다 — 초안(제목/본문/라벨)과 유사 이슈를
   보여준 뒤, 사용자 승인(`y` / 수정 / 기존 이슈에 코멘트)을 받고 나서만 생성한다. 자동 생성 없음.
4. **라벨은 이미 있는 것만.** 새 라벨을 만들지 않는다 (`bug` / `enhancement` / `documentation`
   / `question` / `quality` 등 레포에 존재하는 라벨에서 고른다).
5. Conventional Commits 스타일 제목 (`type: 한국어 요약` — PR 제목 규칙과 동일).

## 절차

### 1. 입력 파악

- `$ARGUMENTS` 가 아이디어/버그 요지. 비어 있으면 **최근 대화**에서 무엇을 남기려는지 유추하고,
  모호하면 한 줄만 물어본다 (긴 인터뷰 금지).
- 기능 제안인지 버그인지 판단한다 (본문/라벨에 반영).

### 2. 맥락 자동 수집

지금 세션의 정보를 모아 이슈에 담을 준비를 한다:

```bash
git remote get-url origin   # 출처 레포 (실패해도 무방)
```

- **출처 레포**: 위 remote 또는 현재 작업 디렉터리.
- **무엇이 트리거했는지**: 최근 대화에서 이 아이디어/버그가 나온 상황을 한두 문장으로.
- **관련 코드/에러**: 방금 본 rocky 관련 코드 경로·에러 메시지·재현 스니펫이 있으면 인용
  (없으면 생략 — 억지로 만들지 않는다).

### 3. 타입 → 라벨 추론

```bash
gh label list --repo minjun0219/rocky
```

- 버그면 `bug`, 기능 제안이면 `enhancement`, 문서면 `documentation`, 질문/미결이면 `question` 등
  **존재하는 라벨에서만** 고른다. 애매하면 라벨 없이 두거나 사용자에게 확인.

### 4. 유사 이슈 조회 (중복 방지)

```bash
gh issue list --repo minjun0219/rocky --state open --search "<핵심 키워드>" --limit 10
```

- 유사한 열린 이슈가 있으면 번호·제목을 보여주고, 새로 만들지 / 기존 이슈에 코멘트로 덧붙일지
  선택지에 포함한다.

### 5. 초안 제시

아래 형태의 초안을 만들어 보여준다 (본문 한국어):

- **제목**: `type: 한국어 요약` (예: `feat: worklog 를 standalone CLI 에도 노출`).
- **본문**:
  ```markdown
  ## 요지
  <아이디어/버그 한두 문장>

  ## 출처
  <출처 레포> 에서 작업 중 발견 (캡처일: <YYYY-MM-DD>)

  ## 맥락
  <무엇이 트리거했는지 / 관련 코드·에러>

  ## 제안 / 재현
  <기능이면 제안, 버그면 재현 절차·기대 동작>
  ```
- **라벨**: 3단계에서 고른 것.

### 6. 확인

사용자에게 `y`(그대로 생성) / 수정(제목·본문·라벨 조정) / `#N 에 코멘트`(기존 이슈에 덧붙임)
중 선택을 받는다.

### 7. 생성

- 새 이슈:
  ```bash
  gh issue create --repo minjun0219/rocky \
    --title "<Conventional 한국어 제목>" \
    --body "<본문>" \
    --label "<라벨>"
  ```
- 기존 이슈에 코멘트:
  ```bash
  gh issue comment <N> --repo minjun0219/rocky --body "<덧붙일 내용>"
  ```
- 생성된 이슈 URL(또는 코멘트 링크)을 출력하고, **원래 작업으로 돌아가도록** 한 줄로 마무리한다.

## 실패 / 예외 처리

- `gh` 미설치/미인증 → `gh auth status` 확인을 안내하고 멈춘다 (초안까지는 보여줄 수 있다).
- `$ARGUMENTS` 도 없고 대화에서도 유추 불가 → 무엇을 남길지 한 줄만 물어본다.
- 사용자가 확인에서 취소 → 아무것도 만들지 않고 멈춘다 (초안만 남긴다).
